## Notable factual corrections (all verified against code)

- **PDF is not "the same renderer as preview".** Four docs claimed a single shared
  drawing path; `ShaftPdfComposer` never calls `ShaftLayout.compute()`/`ShaftRenderer`
  and has its own fit functions. All corrected to the separate-paths fact.
- **`docs/PDF_EXPORT.md` said portrait; the PDF is landscape** (792×612 pt), and the
  "no compression" invariant was contradicted by the shipped round-stock S-break
  feature — both fixed.
- **The NaN/Infinity/>100000 "safety filter" never existed.** Both validation docs
  asserted it; rewritten to the real mechanisms (parse-or-revert + per-field clamps)
  with the missing-upper-bound gap documented.
- **`addThreadAt` parameter order was documented backwards** (pitch before major-Ø) —
  exactly the ~0.5-TPI bug class the doc warns about.
- **Add-dialog default values were an entire generation stale** (100 mm-era values vs
  the 406.4/304.8/152.4 mm reality).
- Thread `tpi` is a persisted model field, not "UI-edge only"; the resolved-component
  pipeline is shipped, not "planned" (3 docs); body keyway is shelved, not "planned"
  (3 docs); the units doc described a v1/v2 envelope scheme that never existed.

## Open questions / potential code issues surfaced (no code changed)

1. **Carousel ordering:** `ComponentsOrdering.md` v1.1 LOCKED newest-on-top, but the
   carousel now displays resolved components in physical order; the ViewModel still
   maintains a newest-first `componentOrder` that display ignores. Doc updated to
   v1.2 describing reality — **needs a product decision** (accept geometric order and
   remove the dangling state, or treat as regression).
2. **Dead code found while verifying docs:** `ShaftSpec.validate()` (never called),
   `RenderOptions.showCenterline` (never read), the duplicate Float-based
   `toMmOrNull`/`parseFractionOrDecimal` pair in `ShaftScreen.kt` vs `util/Parsing.kt`.
3. **No numeric upper bound anywhere** — a finite-but-absurd value (9999999 mm)
   commits successfully (documented as a gap in `VALIDATION_RULES.md` §2.3).
4. **Export gate is narrow:** `blockingExportError()` checks only thread/liner
   positions; taper overlaps warn but never block export (now documented).

## Conventions going forward

- Contracts live in the in-source pack; repo-level `docs/` holds cross-cutting
  references; resolved one-offs go to `docs/archive/`.
- A doc that merely restates a file's API should be a KDoc header instead.
